# 毛球 App — 虚构案例设定

> 本案例用于贯穿整个教程系列。所有 PRD、埋点、SQL、运营策略的示例都基于这个虚构产品。

---

## 产品概述

**毛球**是一个面向宠物主人的移动应用，定位为「宠物主人的内容社区 + 本地生活服务平台」。

核心价值主张：让养宠更简单、更有趣、更省心。

### 核心功能模块

| 模块 | 功能 | 对应的产研场景 |
|------|------|---------------|
| **社区** | 图文/短视频发布、浏览、互动（点赞/评论/收藏） | 内容推荐算法、UGC 运营策略 |
| **附近服务** | 宠物医院、寄养、美容、遛狗，按距离和评分排序 | LBS 服务匹配、商家入驻流程 |
| **AI 问答** | 基于宠物健康知识库的智能问答 | RAG 架构、知识库建设、Prompt 设计 |
| **商城** | 宠物用品推荐、团购、限时折扣 | 推荐算法、转化漏斗、活动系统 |
| **宠物档案** | 记录宠物品种、年龄、疫苗、体重变化 | 用户画像、个性化推荐基础 |

### 用户角色

| 角色 | 描述 | 关键行为 |
|------|------|---------|
| **普通宠物主** | 日常养宠用户，以消费内容和服务为主 | 浏览社区、预约服务、购买用品 |
| **内容创作者** | 活跃的分享者，产出优质宠物内容 | 发布图文/视频、互动、积累粉丝 |
| **服务商家** | 宠物医院、寄养店等线下服务提供方 | 管理店铺信息、接单、回复评价 |
| **平台运营** | 毛球的运营团队 | 策划活动、管理内容、商家运营 |

## 团队设定

一个典型的小型产品团队：

| 角色 | 人数 | 职责 |
|------|------|------|
| 产品经理 | 1 | 需求、PRD、数据分析、项目推进 |
| 前端开发 | 3 | App (React Native) + 管理后台 (React) |
| 后端开发 | 2 | API 服务 + 推荐系统 |
| 算法工程师 | 1 | 推荐模型、NLP（AI 问答） |
| 运营 | 1 | 内容运营 + 活动策划 |
| 设计师 | 1 | UI/UX（外部协作） |

产品经理就是「你」—— 唯一的 PM，需要覆盖从需求到上线的全链路。这正是 AI 工作流最能发挥价值的场景。

## 当前阶段

毛球已完成 MVP（社区 + 附近服务），进入增长阶段。接下来的重点：

1. **AI 推荐系统** — 首页从时间线改为个性化推荐 feed
2. **AI 健康问答** — 接入大模型，基于宠物档案提供个性化回答
3. **运营活动体系** — 搭建活动模板系统，支撑节日活动、拉新、促活
4. **商业化** — 商家推广、品牌合作、会员体系

这四个方向会在教程的不同阶段作为示例出现。

## 数据设定（用于取数和埋点示例）

### 关键指标

| 指标 | 当前值 | 目标 |
|------|--------|------|
| DAU | 12 万 | 20 万 |
| 人均浏览帖子数 | 8 篇/天 | 15 篇/天 |
| 内容发布率（DAU 中发帖占比） | 3.2% | 5% |
| 服务预约转化率 | 1.8% | 3% |
| 次日留存 | 42% | 50% |
| 7 日留存 | 18% | 25% |

### 虚构表结构（SQL 示例用）

```sql
-- 用户表
t_user (user_id, nickname, city, register_time, last_login_time, user_level)

-- 宠物档案
t_pet (pet_id, user_id, pet_name, species, breed, birth_date, weight_kg)

-- 内容表
t_post (post_id, user_id, content_type, title, publish_time, like_count, comment_count, share_count)

-- 服务商家
t_merchant (merchant_id, name, category, city, district, lat, lng, rating, review_count)

-- 服务预约
t_booking (booking_id, user_id, merchant_id, service_type, book_time, status, amount)

-- 用户行为日志
t_event_log (event_id, user_id, event_type, page, element, extra_json, event_time)

-- 推荐曝光
t_recommend_log (log_id, user_id, post_id, position, is_clicked, is_liked, recommend_time)
```
