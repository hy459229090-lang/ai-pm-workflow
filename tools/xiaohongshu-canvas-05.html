Agent Team 实战
一个人管 14 个 AI
是什么体验
从混乱到秩序，我走了 3 个月
14 个 AI 给我干活
我比我们自己干还累
人肉路由器
每个信息都要经过我
每个冲突都要我来裁决
每隔 5 分钟切换一次对话
进度混乱，没人统筹
两版架构对比
❌ 第一版：完全分布式
把 14 个 AI 拉进群，说"你们自己干"。结果一片混乱，我是人肉路由器。
✅ 第二版：主从架构 + 评审机制
设队长统筹全局，定义输入输出，设置评审点，冲突由评审组裁决。我只跟队长对话。
6 个核心角色
队长 Agent
统筹分发，我只跟他对话
产品 Agent
写 PRD、用户故事
技术 Agent
架构设计、API 定义
数据 Agent
埋点方案、看板设计
测试 Agent
测试用例、边界场景
评审 Agent
审查产出，通过/打回
协作流程
1
你 → 队长 Agent：需求描述
2
队长 → 产品 Agent：写 PRD
3
产品 Agent → 评审 Agent：PRD 评审
4
评审通过 → 技术/数据/测试并行工作
5
各 Agent → 队长：交付
6
队长 → 你：最终交付物
什么时候拉 Team？
任务特征 用 Skill 拉 Team
产出数量 1 份 ≥3 份
依赖关系 简单 复杂
你的精力 能覆盖 覆盖不了
一句话：你 review 不过来就拉 Team
