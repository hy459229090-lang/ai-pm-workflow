### P2 - 痛点
**标题**：14 个 AI 同时干活 产出风格完全不同

### P4 - 模板vs规范
**标题**：模板 ≠ 规范
**表格**：
| | 模板 | 规范 |
|---|---|---|
| 回答什么 | 结构长什么样 | 遵守什么规则 |
| 灵活性 | 高 | 低 |
| 例子 | PRD 模板 | 命名规则 |
**结论**：模板是骨架，规范是肌肉记忆

### P5 - 模板体系
**标题**：三个核心模板
**内容**：
- PRD 模板：背景→目标→场景→方案→埋点→排期
- 埋点方案模板：事件列表→属性字典→验证规则
- 数据报告模板：核心指标→趋势分析→洞察建议

### P6 - 规范体系
**标题**：三件套
**内容**：
1. 命名规范：PRD 用「模块_功能_PRD.md」
2. 写作规范：说人话、具体优先、承认不确定
3. 术语规范："用户"≠"账号"，"埋点"≠"日志"

### P7 - 误区
**标题**：常见误区
**错**："模板和规范会限制创造力"
**对**：模板解决低级问题，让你把精力放在真正的创造力上

### P8 - 核心观点
**标题**：模板不是限制 是解放
**内容**：它把不需要创造力的部分自动化了 让你把创造力用在刀刃上

## 小红书正文文案

14 个 AI 同时帮我干活的时候，我遇到了一个新问题：

产出越来越多，风格越来越乱 🤯

毛球推荐 feed 改版，产品 Agent 写的 PRD 是一个风格
技术 Agent 写的召回方案是另一个风格
数据 Agent 出的埋点文档又是第三个风格

前端研发拿到 PRD 说："这个推荐卡片排序逻辑跟以前的 PRD 格式不一样，我没看懂相关性得分怎么计算"
测试拿到埋点方案说："recommend_expose 和 rec_exposure 是一个东西吗？"

同一个项目出来的三份文档，读起来像是三个不同公司写的

后来我才意识到
这不是 AI 的问题，是我的问题

我没有给它们统一的模板和规范
让它们各写各的——那当然不一样

---

**模板和规范，其实是两件事**

模板回答：这份文档应该长什么样？
规范回答：写的时候遵守什么规则？

模板是骨架，规范是肌肉记忆
两者缺一不可

---

**我的模板体系很简单**

📋 PRD 模板
背景 → 目标 → 场景 → 方案 → 埋点 → 排期

📊 埋点方案模板
事件列表 → 属性字典 → 验证规则

📈 数据报告模板
核心指标 → 趋势分析 → 洞察建议

---

**规范三件套**

1️⃣ 命名规范
PRD 用「模块_功能_PRD.md」
不用随意的名字

2️⃣ 写作规范
说人话、具体优先
"毛球社区发布率从 3.2% 涨到 5%"比"用户参与度显著提升"好 10 倍
承认不确定、日期必写

3️⃣ 术语规范
"用户"≠"账号"
"埋点"≠"日志"
"召回"≠"推荐"

把这三样写进 CLAUDE.md
AI 每次都自动遵守

---

**常见误区**

很多人说"模板和规范会限制创造力"

我的经验恰恰相反

模板解决的是格式、命名、术语这些低级问题
这些问题解决了
你才能把精力放在真正需要创造力的地方

没有模板时写 PRD：
30% 想结构 + 20% 调格式 + 50% 写内容

有模板后：
5% 选模板 + 5% 填格式 + 90% 写内容

模板不是限制，是解放 💪

---

**怎么开始？**

不要一开始就搞全套
从你最常写的那份文档开始

想想你过去一个月写最多的文档是什么
从它开始做模板

把你写过的最好的那份同类文档
删掉内容只留结构
这就是你的 1.0 模板
