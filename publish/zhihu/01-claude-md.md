# 如何让 AI 真正理解你的项目？一个文件就够了

> 适合发布为知乎专栏文章，也可回答「怎么让 ChatGPT/Claude 了解我的项目」「AI 编程助手怎么配置」等问题。

---

## 结论

在项目根目录放一个描述项目背景的文件（Claude Code 叫 `CLAUDE.md`，Cursor 叫 `.cursorrules`），写清楚产品定位、团队构成、技术栈、文档结构。50 行，10 分钟。

AI 的产出质量会从"正确的废话"变成"可直接使用的方案"。

---

## 问题在哪

大多数人用 AI 写代码或文档时，交互模式是这样的：

1. 输入一句指令："帮我写推荐算法的 PRD"
2. AI 不知道你的产品是什么、团队多大、技术栈是什么、当前阶段在哪
3. AI 只能输出通用内容："可采用协同过滤或深度学习方案"
4. 你再花 1-2 小时把它改成能用的

每次对话都是"一次性"的——AI 对你的项目没有任何记忆。你要么每次手动描述一遍背景（费时），要么接受通用输出（费力改）。

根本原因：**AI 缺少项目上下文。**

---

## 解法

Claude Code 有一个机制：它在启动时会自动读取项目根目录的 `CLAUDE.md` 文件，作为所有后续对话的背景知识。

你可以把它理解为"给 AI 的项目入职手册"。

### 写什么

| 板块 | 内容 | 作用 |
|------|------|------|
| 项目定位 | 产品是什么、给谁用 | AI 回答的基调 |
| 团队构成 | 人数、角色、各自职责 | AI 判断方案可行性 |
| 技术栈 | 前后端、数据库、部署、AI 服务 | AI 的技术建议能落地 |
| 文档结构 | 目录组织方式 | AI 能自主定位文件 |
| 核心项目索引 | 当前重要项目的关键文档列表 | AI 快速找到相关上下文 |
| 写入规范 | 命名规则、目录归属 | AI 产出符合团队习惯 |

### 写多少

- 刚开始：50-100 行（前三项就够）
- 项目稳定后：100-200 行（加上文档结构和项目索引）
- 上限 300 行（超过的内容拆到其他文件，用导航指令引导 AI 按需读取）

### 效果对比

以一个宠物社区 App 为例（DAU 12 万、1 个算法工程师、React Native 技术栈）。

**没有 CLAUDE.md 时**，问"写推荐算法 PRD"：
> "推荐系统可采用深度学习模型如 DeepFM 进行排序，建议 A/B 测试验证效果。"

问题：1 个算法根本跑不动 DeepFM，12 万 DAU 随机分桶做 A/B 统计检验力不足。

**有 CLAUDE.md 时**，同样的问题：
> "推荐第一版用内容标签 + 协同过滤，冷启动阶段基于宠物档案（品种、年龄）粗排。A/B 测试建议按城市分桶，规避 12 万级 DAU 的样本量问题。"

差距来自：AI 知道了团队规模、数据现状、技术约束。

## 怎么做

### 第一步：创建文件

项目根目录新建 `CLAUDE.md`（大写，这是 Claude Code 的约定文件名）。

### 第二步：填入核心信息

最小可用版本只需要三段：

```markdown
# 项目名称

一两句话的产品简介。

## 团队

| 角色 | 人数 | 职责 |
|------|------|------|
| PM   | 1    | 需求、PRD、数据 |
| 前端 | 3    | App + 后台 |
| 后端 | 2    | API + 推荐 |
| 算法 | 1    | 推荐模型 |

## 技术栈

- 前端：React Native
- 后端：Python (FastAPI)
- 数据库：PostgreSQL + ClickHouse
```

这 20 行就已经能让 AI 的产出质量跳一个台阶。

### 第三步：按需扩展

文档结构、核心项目索引、写入规范——根据你的实际需要逐步补充。

## 其他工具的等价方案

| 工具 | 对应文件 | 说明 |
|------|---------|------|
| Claude Code | `CLAUDE.md` | 启动时自动读取 |
| Cursor | `.cursorrules` | 作用完全等价 |
| GitHub Copilot | 暂无原生支持 | 可通过 prompt 文件间接实现 |
| ChatGPT | Custom Instructions | 贴在系统提示词里 |

核心不是文件名叫什么，而是：**让 AI 在每次交互时都能自动获取你的项目背景。**
