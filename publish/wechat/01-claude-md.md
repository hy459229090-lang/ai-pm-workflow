# 一个文件，让 AI 从"通用助手"变成"项目专家"

## 一个真实的对比

你让 AI 帮你写推荐算法的 PRD。

不给任何背景，AI 回复了 3000 字：
- "根据用户兴趣进行个性化推荐"
- "推荐算法可采用协同过滤或深度学习"
- "建议进行 A/B 测试验证效果"

正确，但全是废话。你还得花 2 小时改成能用的。

现在，你在项目根目录放了一个叫 CLAUDE.md 的文件，把项目背景写进去——产品是宠物社区、DAU 12 万、团队 1 个算法工程师、技术栈是 Python + React Native。

同样的问题，AI 的回答变了：
- "基于内容标签 + 简单协同过滤的两阶段推荐，第一版先做召回，排序后续迭代"
- "冷启动用宠物档案（品种、年龄）做粗排"
- "A/B 测试建议按城市分桶，避免 12 万 DAU 随机分桶样本量不足"

**差距不在模型，在你给了它什么信息。**

---

## CLAUDE.md 是什么

CLAUDE.md 是放在项目根目录的一个 Markdown 文件。Claude Code 每次启动时会自动读取它，作为所有对话的背景知识。

你可以把它理解为**给 AI 的项目入职手册**。一个新同事入职，你会告诉他什么？

- 这个项目做什么的，给谁用
- 团队有哪些人，各自负责什么
- 技术栈是什么
- 文档放在哪里，怎么找
- 有什么规矩和惯例

CLAUDE.md 就是把这些写成文件，让 AI 每次都能看到。**不需要每次对话重复说一遍背景。**

用 Cursor 的同学，等价的东西叫 `.cursorrules`。思路完全一样。

## 写什么、写多少

一个好的 CLAUDE.md 包含这几个板块：

| 板块 | 内容 | 为什么需要 |
|------|------|-----------|
| 项目定位 | 一句话说清楚产品是什么 | AI 回答的基调和语境 |
| 团队构成 | 多少人、什么角色 | AI 评估方案可行性 |
| 技术栈 | 前后端、数据、部署 | AI 技术建议能落地 |
| 文档结构 | 目录怎么组织的 | AI 能自己找文件 |
| 核心项目索引 | 当前最重要的项目和文档 | AI 快速定位上下文 |
| 写入规范 | 命名规则、目录归属 | AI 产出符合团队习惯 |

**不需要全部写满。** 先写前三项（项目定位、团队、技术栈），50 行就够，已经比没有好 10 倍。

控制总长度在 **50-300 行**。太长浪费上下文窗口。超过 300 行的内容，应该拆到其他文件里，用导航指令引导 AI 按需去读。

## 15 分钟动手做

### 第一步：创建文件（1 分钟）

在项目根目录新建 `CLAUDE.md`。

### 第二步：项目定位（2 分钟）

```markdown
# 毛球 — 宠物社区 + 本地服务平台

面向宠物主人的移动应用，核心功能包括宠物日常分享、
附近宠物服务、AI 健康问答和用品团购。

> **AI 导航**：收到问题时请优先读 WORKSPACE_MAP.md
> 定位文档路径，避免全量扫描。
```

### 第三步：团队和技术栈（5 分钟）

```markdown
## 团队

| 角色 | 人数 | 职责 |
|------|------|------|
| 产品经理 | 1 | 需求、PRD、数据分析 |
| 前端 | 3 | App (React Native) + 后台 (React) |
| 后端 | 2 | API 服务 + 推荐系统 |
| 算法 | 1 | 推荐模型、NLP |

## 技术栈

- 前端：React Native + React
- 后端：Python (FastAPI)
- 数据库：PostgreSQL + ClickHouse
```

写到这里，AI 再也不会建议你"用 TensorFlow 训练深度推荐模型"了——它知道你只有 1 个算法工程师。

### 第四步：文档结构 + 核心项目（7 分钟）

把关键目录和当前最重要的文档列出来。这样你说"帮我看一下推荐相关的 PRD"，AI 知道去哪找。

### 验证效果

打开 Claude Code，问一个和你项目相关的问题。感受一下有背景和没背景的差距。

## 常见疑问

**Q：我用 Cursor / GPT 能用吗？**

完全可以。Cursor 用 `.cursorrules`，GPT 可以在 Custom Instructions 里贴项目背景。核心不是文件名，是让 AI 每次都能获取项目上下文。

**Q：需要经常更新吗？**

项目定位和技术栈很少变。核心项目索引可能每周微调。不需要每天更新。

**Q：团队其他人能看到吗？**

CLAUDE.md 就是普通 Markdown 文件，有仓库权限就能看。它同时也是一份给新人的项目速查手册——一石二鸟。
