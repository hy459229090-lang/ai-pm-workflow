# 多平台发布指南

## 平台定位与适配策略

| 维度 | GitHub | 小红书 | 微信公众号 | 知乎 |
|------|--------|--------|-----------|------|
| **核心读者** | 研发、技术型 PM | 泛职场人、好奇型 PM | 产品/研发从业者 | 深度阅读者、技术+产品 |
| **内容调性** | 工具文档，精确实用 | 轻松种草，视觉先行 | 专业长文，观点鲜明 | 分析推演，逻辑严密 |
| **篇幅** | 不限 | 300-800 字 + 图 | 2000-4000 字 | 1500-3000 字 |
| **标题风格** | 描述性 | 抓眼球/数字/疑问 | 观点型/方法论型 | 问题导向 |
| **互动重点** | Star + Fork | 收藏 + 评论 | 在看 + 转发 | 赞同 + 收藏 |
| **引流方向** | — (主阵地) | → GitHub 仓库 | → GitHub 仓库 | → GitHub 仓库 |

## 内容流转规则

```
GitHub (完整版，主阵地)
  ├→ 小红书 (提炼核心观点 + 可视化卡片，种草引流)
  ├→ 公众号 (深度长文版，加案例故事，建立专业形象)
  └→ 知乎 (问答/专栏，强调推演过程，SEO 长尾)
```

**原则**：GitHub 是源头，其他平台是不同剪裁。不为各平台单独创作内容，而是从同一篇教程适配。

## 每篇发布 SOP

1. 在 `docs/` 完成 GitHub 完整版
2. 在 `publish/` 下对应目录生成各平台适配版
3. 文件命名：`00-philosophy.md`、`01-claude-md.md`（与 docs 保持一致）
4. 各平台版本末尾统一带 GitHub 仓库链接
5. 发布顺序：GitHub → 知乎 → 公众号 → 小红书（间隔 2-4 小时，避免同时刷屏）

## 各平台写作模板

### 小红书

```
【标题】用数字/反差/疑问吸引点击（≤20字）

【正文结构】
- 开头：1 句话痛点共鸣（"你是不是也..."）
- 核心：3-5 个要点，每个 1-2 句话
- 收尾：引导收藏 + GitHub 链接

【要求】
- 分段短，每段不超过 3 行
- 可以用 emoji 辅助分隔段落（适度）
- 需要搭配封面图/卡片图（另行制作）
- 300-800 字
```

### 微信公众号

```
【标题】观点型，直接亮立场（≤30字）

【正文结构】
- 导语：场景带入（故事/问题/数据）
- 第一部分：问题是什么（有共鸣的痛点展开）
- 第二部分：方法/思路（核心内容）
- 第三部分：案例演示（毛球场景）
- 结尾：总结 + 下一篇预告 + GitHub 链接

【要求】
- 2000-4000 字
- 段落间空行分隔，适合手机阅读
- 专业但不学术，有观点有温度
- 关键句加粗
```

### 知乎

```
【标题】问题式 或 方法论式（适合搜索）

【正文结构】
- 先给结论（1-3 句话直接回答）
- 展开推演（为什么这样、怎么得出的）
- 实操细节（案例 + 代码/配置片段）
- 总结 + GitHub 链接

【要求】
- 1500-3000 字
- 逻辑链清晰，每一步有因果
- 可以贴代码块
- 适合同时发专栏 + 回答相关问题
```
